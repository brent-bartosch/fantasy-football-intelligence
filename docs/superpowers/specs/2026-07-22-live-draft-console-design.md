# Live Draft Console — design (2026-07-22, user-approved)

## Problem
FP mock drafts are not Yahoo-API-visible, so the only live tool is manual —
and the last mock routed every pick through an LLM chat loop (~90s/response).
Architecture bright line (project record): the pick-time number is computed by
CODE, never by a model. The console removes the model from the pick loop.

## Shape
Single self-contained HTML file, generated by a build script (upgrade path of
`cheat_sheet_html.py`; new builder `scripts/draft_console.py` acceptable).
No server, no network, no LLM at pick time. Millisecond re-ranking.

## Data baked at build time (from the live DB / repo, with provenance)
- Live valuation (players, pos, proj, vorp — starts-based v2), snapshot id.
- `data/p_starts.json` table + `_meta` (fail-loud on mode mismatch at build).
- `DEPLOYED_PARAMS` fields the JS engine needs: pstart weights path,
  `qb_by_round=(2,5,14)`, TE≤2 insurance cap, `defk_round`, feasibility
  constants, roster shape (2QB/2RB/3WR/1TE/1FLEX/1K/1DEF/8BN, 19 rounds).
- `adp_2qb` per player (999 = undrafted sentinel; K/DEF have no ADP).

## JS engine (port of the deployed A′ pick path)
- Score = `P_start[pos][k+1] × vorp` with k = MY current count at pos;
  slots beyond table → weight 0 (never-start candidates dropped).
- Rules: QB timing force by round; TE≤2; DEF/K at `defk_round` (excluded
  from voluntary suggestions before that); feasibility (roster must remain
  fillable in remaining rounds).
- UI state drives it: crossed-off players are out of the pool; "mine" picks
  advance my counts + round.

## UI
- Setup: my draft slot (1–12) prompt up front; snake order derived.
- Board: existing position-column layout, tier colors, search box (keep).
- Actions per player: click = drafted-by-other (gray out); explicit second
  action (e.g. long-press/button) = MY pick (adds to my roster panel).
- Suggestion panel (always visible): top-5 by live engine score with the
  score shown, current round + picks until my next turn, and a "likely gone
  by your next pick" flag (adp_2qb ≤ current overall pick + picks-to-next).
- Undo (mis-clicks happen mid-draft) and a full-board reset with confirm.
- Export button → JSON of the completed draft (pick order, player, team=mine
  /other) in a format `grade_board.py` accepts (read grade_board's input
  format and match it; if it needs a small loader addition, add it there).

## Drift guard (fail-loud, mandatory)
Build script runs the PYTHON deployed engine over a scripted pick sequence
and embeds the resulting golden trace (per-pick top-5 + scores) in the page.
On load, the JS engine replays the same sequence and compares: green badge
if identical, RED badge + banner ("do not trust — engine drift") on any
mismatch. A pytest covers the Python-side trace generation; the browser
self-test covers the JS side. No silent fallbacks anywhere.

## Test loop with a real draft
Build console → user runs an FP mock clicking picks live → export JSON →
`grade_board.py` grade (Spearman edge, same as mock-2) = first live-fire
test of the v2 valuation against a human room.

## Out of scope
The Aug 29 Yahoo draft (draft_assistant.py live mode, auto-polls, hands-off).
Full VONA (ADP-gone flag is the light stand-in). Any model-in-the-loop step.
