# Fantasy Football Draft Intelligence System — 2026 Rebuild
**Design doc — 2026-07-08**
**Status: validated in brainstorming; pending risk-analysis + ADR + implementation plan**

## 1. Context

- **League:** Yahoo, 12-team, head-to-head redraft. Renames annually (2025: "NAJEE 'LEFT EYE' HARRIS", ID 326814); the same core group of ~11 managers spans a 17-year history (2009–2025) already imported into local Postgres (3,782 draft picks, 244 team-seasons). League seasons are linked via the Yahoo API `renew`/`renewed` chain. User's manager profile: **"Sports"** (joined ~4 seasons ago).
- **Scoring is deeply custom** (ground truth: `league_rules.md`): 2 starting QBs, 6-pt passing TDs, full PPR, ±0.5 per completion/incompletion, +0.33 per rush attempt, **+1 per rushing/receiving first down**, tiered yardage bonuses (100/150/200 rush & rec; 300/400/500 pass), return yards, enhanced DEF scoring. No keepers. Draft ≈ mid-August 2026 (~6 weeks out); 90-second pick clock, live snake draft.
- **Prior codebase (2025)** is dormant. Keep: Postgres schema + 17y data, Yahoo OAuth plumbing, `league_rules.md`. Discard: flat-multiplier scoring adjuster, keyword "sentiment analysis," keyword-only "RAG."
- **User priorities:** (1) the draft decides 65–75% of the season — it's the focus; (2) in-season next-man-up/waiver intelligence is earmarked, built after the draft; (3) trades explicitly deprioritized.
- **Budget:** FantasyPros subscription in hand; ≤ ~$100/mo total. Research shows most needs are met free.

## 2. Goals / Non-goals

**Goals**
1. A **value board** that re-scores stat-level projections under the exact league rules and prices 2QB scarcity correctly — the core edge no commercial tool can replicate.
2. A **draft-day assistant** (CLI) that auto-tracks picks from Yahoo, knows roster holes, and recommends instantly, rehearsed to reliability.
3. An **automated morning briefing** that keeps the board fresh (news → audited signals → capped board adjustments) between now and draft day.
4. A **testing ladder** (sim farm → backtests → FP mocks → Yahoo rehearsals → user-in-loop) so the tool has drafted tens of thousands of times before it matters.
5. **Historical mining** of the 17-year league data for descriptive manager tendencies feeding opponent models.

**Non-goals (for this build)**
- In-season lineup/waiver module — designed-for (schema, data feeds accumulate) but built post-draft.
- Trade analysis — earmarked only.
- Web UI / mobile — CLI + generated reports only.
- Fitted ML models on league history — descriptive statistics only (3,782 picks is too small to fit without overfitting).

## 3. Architecture: deterministic spine, agentic edges

**Bright line:** *the number that ranks players at pick time is computed by code, never generated by a model.* LLM agents operate everywhere latency and auditability permit: news digestion, expert-content analysis, briefing authorship, and an advisory annotation channel during the draft. Agents influence the board only **before** decision time, through typed, capped, audited adjustments.

Evidence basis: Amazon's production NFL Fantasy AI paper and every surveyed working hobbyist system converge on this split; systems that let the model do valuation at decision time failed (hallucinated availability, 10–15s latency, untestability).

```
                    ┌─ INGESTION (daily, cron) ─────────────────────────┐
Sleeper projections ┤  FantasyPros API (ECR/ADP)   nflverse actuals     │
Yahoo league data   ┤  Expert RSS/podcast feeds → agent digestion       │
                    └───────────────┬───────────────────────────────────┘
                                    ▼
                    ┌─ CORE ENGINE (deterministic Python + Postgres) ───┐
                    │ scoring engine → projections under league rules   │
                    │ (incl. distribution-based threshold bonuses,      │
                    │  first-down imputation for FD-less sources)       │
                    │ valuation: VORP (2QB baseline) · VONA · GMM tiers │
                    │ opponent models: ADP-noise + manager tendencies   │
                    │ Monte Carlo draft simulator + backtest harness    │
                    └───────────────┬───────────────────────────────────┘
                                    ▼
      ┌─ OUTPUTS ──────────────────────────────────────────────────────┐
      │ morning briefing (agent-written, human-confirmed adjustments)  │
      │ draft-day assistant CLI (Yahoo poll + manual fallback;         │
      │   board decides, async agent annotates between picks)          │
      │ sim-farm nightly adversarial report                            │
      └────────────────────────────────────────────────────────────────┘
```

## 4. Components

### 4.1 Data layer (Postgres, extends existing schema)
| Source | What | Access | Notes |
|---|---|---|---|
| **Sleeper API** | Stat-level projections incl. **first downs**, completions/incompletions, rush att, PR yards, bonus flags; trending adds/drops; historical actuals | Free, no auth, undocumented endpoints | **Primary projection backbone.** Snapshot daily; fail loud on schema drift |
| **FantasyPros public API v2** | ECR (+min/max/std), ADP (incl. superflex), stat-level projections (no FD/targets), news, injuries | Free key (apply at secure.fantasypros.com/api-keys/request); 100 calls/day, 1/sec | Consensus overlay + ADP for opponent models. Cache aggressively |
| **nflverse (nflreadpy)** | Historical weekly actuals: first downs, completions, return yards, snap counts | Free | Ground truth for backtests + FD-imputation regressions |
| **Yahoo Fantasy API** | League history via `renew` chain, draft results (live-capable), weekly stats, transactions | Existing OAuth app; undocumented rate limits (error 999 ≈ 10–15 min lockout) | Import 2025 season + weekly performance for all 17 seasons (throttled, cached) |
| **Expert feeds** | Accuracy-verified shortlist (Koerner, Ratcliffe, Boone, Thorman, Draft Sharks, Bond, Zachariason, Gretch, McFarland, Fantasy Points injury desk) | Verified RSS/podcast URLs where they exist; scrape/email-bridge otherwise | Feeds agent digestion layer |
| Optional paid | 4for4 (only paid source with projected first downs) | ~$ | Add only if FD imputation proves noisy |

Player identity: adopt a cross-source ID crosswalk (dynastyprocess `ff_playerids` / nflverse IDs) mapping Sleeper ↔ FantasyPros ↔ Yahoo ↔ nflverse. This replaces the old name-variation table.

### 4.2 Scoring engine
- Input: per-player stat-line projections (any source, normalized). Output: expected fantasy points under `league_rules.md`, encoded once as versioned config.
- **Threshold bonuses computed on distributions, not means** (a mean projection cannot price a 100/150/200-yd bonus): per-player yardage distributions estimated from historical week-level variance (nflverse) around the projected mean; bonus EV = P(threshold) × bonus. ESPN hidden API threshold-game counts and Sleeper historical bonus flags serve as calibration priors.
- **First-down imputation** for sources lacking FD: regression fitted on nflverse actuals (FD per carry by rusher profile; FD per reception by aDOT/position), applied to projected volume. Validated against Sleeper's native FD projections.
- Same engine scores historical actuals → true historical points under league rules (powers backtests and "what actually won" analysis).

### 4.3 Valuation layer
- **VORP** with 2QB-correct replacement baselines derived from league shape (12 teams × 2QB + flex ⇒ QB replacement ≈ QB20–24, computed not assumed).
- **VONA** at draft time: value vs. best same-position player likely available at next pick, from availability simulation.
- **Tiers:** GMM clustering (Boris Chen's open fftiers method) re-run on FP superflex ECR + our own adjusted values — the user's "cohorts," computed.
- Uncertainty bands carried through from projection spread (ECR std, cross-source disagreement).

### 4.4 Opponent & manager models
- Descriptive tendencies per manager from 17y history (position-by-round propensities, reach patterns, QB timing), anchored to stable manager GUIDs via the renew chain; verify "Sports" continuity and the 14→12 team transition during data audit.
- Draft pick models: softmax/noise draws over superflex ADP, blended with per-manager tendency priors. Used by the simulator and by live VONA.

### 4.5 Monte Carlo draft simulator + testing ladder
- Simulator: full 12-team × 20-round snake in milliseconds; our seat driven by the board (strategy-parameterized), others by opponent models.
- **Level 0 — sim farm:** thousands of drafts nightly; grid over strategy knobs (QB timing, tier-break rules, positional caps). Output: win-rate deltas with confidence intervals via roster season-scoring.
- **Level 0.5 — backtests:** draft 2023–2025 with that year's preseason projections/ADP; score rosters with **actual** results (nflverse) under league scoring. Breaks projection circularity.
- **Level 1 — FP Draft Wizard browser mocks:** 5–10/day batches via browser automation (independent bot opponents, UI-flow sanity). Not 24/7 — anti-bot/ToS risk.
- **Level 2 — Yahoo dress rehearsals:** private test league; drill OAuth refresh, poll lag, 999 backoff, manual-fallback switchover.
- **Level 3 — user-in-loop mocks:** user drafts with assistant advising; log every human override as a bug-or-insight.
- **Level 4 — the real draft.**
- Nightly sim report is **adversarial**: worst drafts, failure clusters, assumption audits — not just win rates.

### 4.6 Draft-day assistant (CLI)
- **Pick tracking:** poll Yahoo `draftresults` every 5–10s (entries without `player_key` = unmade picks; diff for new). Proactive OAuth refresh. On error 999: exponential backoff + **automatic switch to manual-entry mode** (single keystroke pick logging, practiced in rehearsals). Static printed board as final floor.
- **Recommendation:** instant, from precomputed board + deterministic roster-need logic + live VONA. Reproducible; no model in the loop.
- **Async agent lane:** between user's picks (~15-min windows), agent re-runs availability sims, checks breaking news against the next-pick window, annotates top board options with reasoning. Advisory only; if unfinished when on the clock, board answer stands alone.

### 4.7 Morning briefing pipeline (cron, from week 2)
- Ingest feeds → agent digests into **schema-validated structured signals**: `{player_id, signal_type (injury|role_change|depth_chart|hype), direction, magnitude, source, confidence, evidence_url}`.
- Signals apply to the board only through **typed, capped adjustments** (projection nudge capped at ±10% per player per day and ±20% cumulative from signal provenance; injury/role flags are unbounded but purely informational); every adjustment logged with provenance; user confirms via the briefing before adjustments become active. Unconfirmed signals display but don't move numbers.
- Briefing content: ranking moves, new signals, feed health, sim-farm findings, ADP drift.

### 4.8 Error handling (fail-loud, per user preference)
- Any ingestion failure, schema drift, stale snapshot, or crosswalk miss → prominent briefing alert + refusal to silently serve stale/partial derived outputs (outputs carry data-vintage stamps).
- Draft-day: degraded modes are explicit and announced in-UI (LIVE → POLL-DEGRADED → MANUAL → PAPER), never silent.

### 4.9 In-season module (earmarked, post-draft)
Data accumulation starts now (snap counts, transactions, usage); build after draft: next-man-up detection (depth-chart + usage deltas + injury signals), waiver recommendations under 65-move/5-per-week budget, start/sit education layer citing Boone/tier logic, Subvertadown-style DEF/K streaming. Trades: awareness only.

## 5. Build order (~6 weeks)
1. **Wk 1:** infra revival (Postgres up, data audit, manager-continuity verification); Yahoo renew-chain import incl. 2025 season + weekly stats; FP API key application (lead time unknown — apply day 1); Sleeper + nflverse ingestion; ID crosswalk.

## 6. Testing strategy
- **Golden tests:** scoring engine output vs. Yahoo's official computed points for known 2025 player-weeks (exact-match requirement).
- **Property tests:** valuation monotonicity, replacement-baseline sensitivity.
- **Backtests as regression suite:** strategy changes must not degrade 2023–2025 backtest outcomes.
- **Rehearsal drills as acceptance tests** for draft-day plumbing (documented runbook: token refresh, 999 backoff, manual switchover).
- Agent-layer audits: every signal traceable to source; sampled human review via briefing confirm gate.

## 7. Open questions (carried into risk analysis)
1. Yahoo live-draft poll lag is unmeasured (no published numbers) — measure in Level 2 rehearsal.
2. Sleeper undocumented endpoints could drift — snapshot + fail-loud mitigates; 4for4 is the paid fallback.
3. FP API key approval is discretionary — apply immediately; `ffpros`-style page parsing is the fallback.
4. 2026 league settings could change (roster/scoring tweaks happen annually) — re-verify rules when Yahoo opens the 2026 league; scoring config is versioned.
5. Historical projection/ADP archives for backtests (2023–25 preseason) need sourcing (candidates: dynastyprocess ADP archive, ffanalytics raw archives, Wayback FP pages).
